# 粉丝活动商品核销平台 PRD（产品文档）

## 1. 项目背景

本平台是一个面向偶像团体粉丝活动的线上商城系统，用于售卖粉丝向商品或服务，例如演出物料、特典商品、相片或数字内容以及线下活动物贩商品。

用户在平台购买的通常是**商品凭证或数字资产**。部分商品需要在**线下活动现场由工作人员核销后领取实物**。

平台需要支持： - 商品购买 - 电子钱包 - 商家代金券 - 线下核销 - 商家统计

------------------------------------------------------------------------

## 2. 系统角色

### 用户

-   浏览商家与商品
-   购买商品
-   充值电子货币
-   出示核销二维码
-   查看订单与核销记录

### 商家

-   创建商品
-   查看订单
-   查看销售数据
-   查看核销记录

### 核销工作人员

-   扫描用户二维码
-   选择核销商品
-   确认核销

### 平台管理员

-   管理商家
-   管理活动
-   管理平台数据

### 商家代金券

## 8. 用户核销二维码

二维码绑定 **用户账户**，而不是订单。

原因：

-   用户可能多次购买
-   可以跨订单核销
-   核销按商家统计

推荐使用 **动态二维码**。

二维码数据示例：

token\
user_id\
timestamp\
signature

有效期建议：**3\~5分钟**。

------------------------------------------------------------------------

## 9. 核销流程

``` mermaid
flowchart TD
A[用户出示二维码]
A --> B[工作人员扫码]
B --> C[系统解析二维码]
C --> D[获取用户ID]

D --> E[查询用户在该商家的资产]

E --> F[工作人员选择核销商品]

F --> G[核销预览]

G --> H[确认核销]

H --> I[扣减商品数量]

I --> J[生成核销记录]
```

------------------------------------------------------------------------

## 10. 核销安全设计

防止重复核销需要：

-   动态二维码
-   数据库事务
-   资产锁

``` mermaid
flowchart TD
A[扫码二维码]
A --> B[验证Token]

B --> C{Token是否有效}

C -->|否| D[拒绝核销]

C -->|是| E[查询用户资产]

E --> F[选择核销商品]

F --> G[系统加锁]

G --> H{资产是否充足}

H -->|否| I[核销失败]

H -->|是| J[扣减资产]

J --> K[生成核销记录]
```

## 13. 核销点管理

建议支持多个核销点：

    商家A
     ├ 核销点1
     ├ 核销点2
     └ 核销点3

每个核销点：

-   工作人员账号
-   核销权限
-   核销记录

## 15. 平台总结

核心业务流程：

用户购买商品\
↓\
订单系统处理\
↓\
支付系统完成支付\
↓\
资产系统生成代金券\
↓\
用户线下活动核销\
↓\
核销系统记录数据\
↓\
商家统计销售

适用场景：

-   粉丝见面会
-   偶像周边物贩
-   粉丝市集
-   线下应援活动
